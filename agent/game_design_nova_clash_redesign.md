# Game Design Draft — Nova 1492 x Clash Royale 코옵 PvE

이 문서는 차기 게임 방향을 위한 **리디자인 초안**이다.  
현재 SSOT인 `agent/game_design.md`를 바로 대체하지 않고, 새 방향을 별도 초안으로 정리한다.

## 한 줄 정의

매치 전에 조립한 3~5기 유닛 팀을 들고 들어가, 각자 에너지로 유닛을 소환해 공용 코어를 지키는 모바일 우선 코옵 PvE 빌드 디펜스.

## 핵심 컨셉

| 항목 | 값 |
|---|---|
| 장르 | 코옵 PvE 빌드 디펜스 |
| 플랫폼 전략 | 모바일 우선 |
| 세션 길이 | 5분 내외 |
| 플레이 인원 | 2~4인 |
| 첫 완성 모드 | 코옵 PvE |
| 전장 구조 | 반개방 |
| 시야 | 전체 공개 |
| 전투 개입 | 유닛 소환 중심 |
| 전투 자원 | 각자 에너지 |
| 메타 구조 | 가벼운 차고 |

## 무엇을 가져오고 무엇을 버리는가

### Nova 1492에서 가져오는 것

- 사전 편성의 비중
- 유닛 조립과 모듈 조합
- 유닛별 성격 차이
- 준비 단계에서 머리 쓰는 재미

### Clash Royale에서 가져오는 것

- 전투 중 슬롯 기반 유닛 소환
- 시간이 지나며 차는 에너지
- 짧고 읽히는 전투 템포
- 모바일 친화 입력

### 이번 방향에서 버리는 것

- 직접 캐릭터 조작
- 플레이어 점프/이동 기반 액션
- 프렌들리 파이어 중심 설계
- 다운/구조 중심 코옵 드라마
- 노바식 시야 제한
- 런 중 드래프트 중심 구조
- 웨이브 사이 보상 선택 중심 성장 루프

## 코어 재미

- **준비 단계의 빌드 재미**: 유닛 프레임과 모듈 조합으로 팀 편성을 짜는 시간이 전투만큼 중요해야 한다.
- **짧고 읽히는 전투 템포**: 전투 중 판단은 복잡한 조작이 아니라, 어떤 유닛을 언제 어디에 내는가에서 나온다.
- **서로 다른 편성의 협력**: 각자 다른 조합을 들고 와 공용 코어를 함께 지키는 과정에서 코옵 감각이 생겨야 한다.
- **배치가 만드는 전략**: 숨겨진 정보 대신 지형, 진입 경로, 소환 위치, 앵커 반경이 전술 판단을 만든다.

## 한 판 구조

1. 로비에 입장한다.
2. 각자 차고에서 유닛 3~5기와 모듈 조합을 편성한다.
3. 전투에 진입한다.
4. 전투 중 각자 에너지로 자기 유닛 슬롯에서 유닛을 소환한다.
5. 팀이 공용 코어를 지키며 연속 압박을 버틴다.
6. 성공 또는 실패 결과 화면으로 종료한다.

핵심은 `런 중 성장`이 아니라 `매치 전 편성`이다.  
전투는 준비 결과를 검증하는 짧은 시험장이어야 한다.

## 플레이어 역할

- 플레이어는 직접 조작 캐릭터가 아니다.
- 플레이어는 `차고 소유자`, `전투 중 소환자`, `빌드 책임자`다.
- 전투 중 플레이어의 기본 액션은 다음 네 가지다.
  - 유닛 슬롯 확인
  - 유닛 선택
  - 배치 가능 구역 지정
  - 소환 타이밍 판단

즉 플레이어 판타지는 `아바타 액션`이 아니라 `현장 지휘`에 가깝다.

## 차고와 편성

### 기본 원칙

- 한 플레이어는 3~5기 소수 정예 유닛을 편성한다.
- 각 유닛은 `기본 프레임 + 모듈 조합`으로 성격이 달라진다.
- 빌드 재미는 `유닛을 얼마나 많이 모았는가`보다 `어떤 프레임에 어떤 모듈을 얹었는가`에서 나와야 한다.

### 메타 구조

- 메타는 `가벼운 차고`를 기본으로 둔다.
- 기본 유닛과 기본 모듈은 너무 강하게 잠그지 않는다.
- 플레이를 통해 차고 폭, 선택지, 모듈 종류가 점점 넓어지는 정도로 둔다.
- 카드 수집 중심 메타나 과한 등급 강제 구조는 첫 방향에서 제외한다.

### 유닛 차이의 출처

유닛 차이는 복잡한 AI보다 아래 조합에서 나온다.

- 프레임 체급
- 이동성
- 앵커 반경
- 공격 방식
- 사거리
- 유지력
- 모듈 시너지

## 전장 구조

- 전장은 반개방으로 둔다.
- 시야는 전체 공개다.
- 맵 재미는 숨겨진 정보가 아니라 진입 경로, 길목, 후방 안전지대, 코어 주변 공간에서 만든다.
- 완전 자유 배치보다는 `읽기 쉬운 배치 구역`을 둔다.
- 첫 버전은 맵 볼륨보다 가독성을 우선한다.

전장의 역할은 “적을 못 보게 만드는 것”이 아니라,  
**어디에 어떤 유닛을 배치해야 효율적으로 막히는가를 묻게 만드는 것**이다.

## 전투 규칙

### 입력과 소환

- 플레이어는 자기 유닛 슬롯을 눌러 유닛을 선택한다.
- 배치 가능 구역 안에서 유닛을 소환한다.
- 소환 비용은 각자 에너지를 사용한다.
- 에너지는 전투 중 시간에 따라 회복된다.
- 전투 중 기본 판단은 `무엇을`, `언제`, `어디에` 내는가다.

### 협동 자원

- 코어는 팀 공용 목표다.
- 에너지는 각자 따로 쓴다.
- 공용 자원 분배 대신, 공용 위협 대응이 코옵의 중심이어야 한다.

즉 협력은 “에너지를 나눠 쓰는 것”이 아니라,  
“누가 어떤 압박을 맡을지, 어떤 조합으로 서로의 약점을 메울지”에서 나온다.

### 유닛 행동

- 유닛은 소환 위치에 자기 `앵커`를 가진다.
- 유닛은 자기 `앵커 반경` 안에서만 이동하고 교전한다.
- 앵커 반경은 전역 고정값이 아니라, 유닛 스펙과 모듈 조합 결과로 달라진다.
- 따라서 역할 차이는 고급 AI보다 빌드 스펙에서 나온다.

예시:
- 탱커: 좁은 앵커 반경, 높은 유지력
- 헌터: 넓은 앵커 반경, 적 추적 성향
- 포격형: 매우 좁은 앵커 반경, 높은 사거리
- 지원형: 중간 앵커 반경, 보조 능력

### 첫 버전에서 하지 않을 것

- 전투 중 유닛 재배치
- 순찰점 여러 개 지정
- 수동 웨이포인트 지정
- 고급 역할 AI
- 플레이어 수동 시전 비중이 큰 액션 구조

첫 버전은 `앵커 반경 기반 자동 교전`만으로도 전략적이어야 한다.

## 적과 압박 구조

- 적은 코어를 향해 연속 압박을 준다.
- 플레이어는 압박에 맞춰 유닛 순서를 꺼내며 대응한다.
- 적 재미는 화려한 패턴보다 `무슨 조합으로 들어오느냐`에서 시작한다.

초기 적 역할 예시:
- 러셔: 빠르게 코어로 뛰는 적
- 탱커: 전선을 오래 끄는 적
- 후열 위협형: 원거리 유닛을 압박하는 적
- 군집형: 광역 대응을 강요하는 적

## 코옵 역할 분화

명시적 클래스는 없다.  
**편성이 역할을 만든다.**

예시:
- 한 명은 전방 유지형
- 한 명은 후방 화력형
- 한 명은 광역 정리형
- 한 명은 지원/교란형

좋은 코옵 상태는 “같이 한 전장을 보고 있는데, 각자 잘하는 대응이 다르다”는 느낌이 드는 상태다.

## MVP 범위

### 포함

- 코옵 PvE 한 모드
- 반개방 맵 1종
- 공용 코어 방어
- 각자 에너지
- 유닛 슬롯 기반 소환
- 앵커 반경 기반 자동 교전
- 3~5기 편성
- 기본 차고 구조
- 적 역할 3~4종

### 제외

- PvP
- 전투 중 재배치
- 시야 제한
- 런 중 드래프트/보상 선택
- 프렌들리 파이어
- 다운/구조
- 수동 경로 지정
- 복잡한 메타 화폐/강화 트리

## 재미 가설

이 게임의 방향은 아래 세 질문으로 검증한다.

1. 매치 전에 편성하는 시간이 전투만큼 기대되는가
2. 전투 중 어떤 유닛을 먼저 꺼낼지 고민하게 되는가
3. 한 판이 끝난 뒤 다음엔 어떤 조합으로 바꿔볼지 떠오르는가

이 셋이 살아 있으면 방향이 맞다.

## MVP 검증 포인트

- 플레이어가 첫 1분 안에 “나는 캐릭터를 움직이는 게 아니라 유닛을 소환하는 플레이어”라는 점을 이해할 것
- 유닛 슬롯과 에너지 규칙이 짧은 설명만으로도 이해될 것
- 같은 맵에서 편성만 바꿔도 대응 방식 차이가 체감될 것
- 코옵 한 판 후 서로의 편성 이야기가 자연스럽게 나올 것
- 반개방 맵이 복잡하지 않고 읽히는 수준을 유지할 것

## 기존 레포에서 재사용 가능한 것

- Photon 로비와 룸 시작 구조
- CoreObjective 방어 구조
- Wave와 Enemy 스폰 파이프라인의 기본 뼈대
- 일부 스킬 프리젠테이션 자산의 재해석 가능성

## 장기적으로 크게 바뀌는 것

- Player: 직접 조작 아바타에서 차고 소유자/전투 중 소환자로 재정의
- Skill: 직접 시전 스킬에서 유닛 모듈/유닛 능력 중심으로 재정의
- Combat: 플레이어 액션 판정 중심에서 유닛 소환과 앵커 반경 교전 중심으로 재정의
- Wave: 웨이브 사이 보상 선택 중심 구조에서 연속 압박 기반 구조로 재정의
