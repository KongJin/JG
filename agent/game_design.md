# Game Design — 카오스 코옵

2~4인이 5~10분 동안 웨이브를 버티는 카오스 코옵 생존 게임.
개수·범위·지속 성장과 덱 순환형 스킬 운용으로 빌드를 만든다.
협력은 필수지만, 친구의 스킬도 절반만큼은 아프다.

## 핵심 컨셉

| 항목 | 값 |
|---|---|
| 한 줄 정의 | 친구와 함께 살아남아야 하지만, 친구의 스킬도 위험한 협력 생존 게임 |
| 세션 길이 | 5~10분 |
| 플레이 인원 | 2~4인 |
| 목표 | 웨이브 생존 |
| 플랫폼 전략 | PC 우선, 이후 모바일 확장 고려 |
| 장르 표기 | 코옵 아레나 서바이벌 로그라이트 + 프렌들리 파이어 |
| 브랜딩 | 카오스 코옵 |

## 코어 재미

- **혼돈 속 협력**: 함께 적을 막아야 하지만, 아군 공격도 피해로 들어와 위치 선정이 중요하다
- **빌드 성장 체감**: 개수, 범위, 지속이 커질수록 스킬이 화면에서 즉시 강해진다
- **즉흥적 판단**: 조준 보조 기반으로 조작은 단순하지만, 어떤 스킬을 언제 쓸지와 전장 포지셔닝이 깊이를 만든다

## 전투 규칙

| 규칙 | 내용 |
|---|---|
| 프렌들리 파이어 | 아군 피해 50% 감쇠 (후반 스케일링 시 30%로 추가 감쇠 검토) |
| 자기 자신 피해 | 없음 |
| 조준 | 방향은 수동, 타겟팅 보조만 자동 (마우스/터치 포인트 기준 가까운 적 우선) |

### 조준 설계 원칙

자동 조준이 AoE 방향/위치까지 자동으로 잡으면 프렌들리 파이어가 "내 실수"가 아니라 "시스템 탓"이 된다.
**조준 방향은 플레이어가 결정**해야 프렌들리 파이어에 의미가 생긴다.

### 프렌들리 파이어 스케일링 주의

개수 성장으로 투사체가 늘어날수록 아군 피해도 비례해서 늘어난다.
4인 풀빌드 후반에 아군 피해가 적 피해를 넘으면, 플레이어들이 서로 멀리 떨어져 코옵 느낌이 줄어든다.
→ 후반 감쇠 비율 조정, 또는 근접 범위 아군 피해 감소 버프 등 **뭉칠 인센티브**를 함께 설계할 것.

## 스킬/자원 시스템 — 덱 순환

- 스킬 획득 시 **미사용 카드 더미**로 들어간다
- 현재 슬롯의 스킬을 사용하면 → **사용한 카드 더미**로 이동
- 미사용 카드 더미 맨 앞 카드가 빈 슬롯을 채운다
- 미사용 카드 더미가 비면, 사용한 카드 더미와 교체한다

이 게임의 MVP 정체성은 전통적인 조합형 덱빌딩보다
**로테이션 기반 로드아웃 드래프팅**에 가깝다.
의미 있는 판단은 "어떤 시너지를 완성하느냐"보다
"새 스킬을 넣어 대응 범위를 넓힐지, 기존 스킬을 강화해 순환 품질을 높일지"에서 나온다.

### 실시간 UX 주의

덱빌더는 보통 턴제(Slay the Spire)라서 카드 순서를 생각할 시간이 있다.
실시간 카오스 전투에서는 판단 여유가 제한되므로, 슬롯 수를 2~3개로 제한해서
"지금 쓸지 아끼고 다음 걸 쓸지" 정도의 **가벼운 판단**으로 만드는 것이 적합하다.
→ 프로토타입에서 덱 순환 실시간 UX를 가장 먼저 검증할 것.

## 성장 구조

### 공용 성장축 (MVP)

| 축 | 예시 |
|---|---|
| 개수 | 투사체 수, 동시 생성 수 |
| 범위 | 반경, 판정 크기, 폭발 범위 |
| 지속 | 존 시간, 효과 시간, 생성물 유지 시간 |
| 안전 | 아군 피해 억제, 포지셔닝 안정성, 구조 상황 유틸 |

### 성장축 설계 원칙

- MVP에는 **경험치/레벨업 시스템을 넣지 않는다**. 웨이브 클리어 자체가 성장 트리거 역할을 한다.
- 메인 성장은 **웨이브 종료 후 3지선다**로 제공한다.
- 선택지는 **새 스킬 추가** 또는 **보유 스킬 강화**를 섞어서 제시한다.
- **필드 드롭은 영구 성장에 사용하지 않는다.** 힐, 짧은 버프, 즉시 폭발 같은 전술 자원만 담당한다.
- **회전율은 덱 순환이 전담**한다. MVP 업그레이드 축에 Cooldown은 넣지 않는다.
- 네 개의 성장축을 모든 스킬에 전부 열지 않는다. 각 스킬은 자기 판타지에 맞는 **2~3개 축만 의미 있게 개방**한다.

### Safety 축 원칙

Safety는 단순 방어 스탯이 아니다.
"아군 피해 -20%" 같은 무조건 이득형 수치 옵션은 세금 선택지가 되기 쉽다.
Safety는 반드시 **공격성, 범위, 운용 편의 중 하나와 교환되는 선택지**로 설계한다.

예시:
- 폭발 범위 감소 + 아군 피해 감소
- 시전 순간 짧은 자기 보호막 부여
- 아군 근처에서는 위력 감소 대신 안전성 증가
- 다운된 아군 근처에서만 구조/생존 관련 추가 효과 부여

이 원칙을 통해 Safety는 "소극적 방어"가 아니라
팀 플레이 방식과 포지셔닝을 바꾸는 별도의 강함으로 기능해야 한다.

### 성장 획득 방식

MVP에서는 **웨이브 종료 후 스킬 3지선다**를 메인 성장 루프로 사용한다.
선택한 스킬은 즉시 슬롯을 덮어쓰지 않고, **미사용 카드 더미에 편입**되어 이후 덱 순환으로 등장한다.
개수/범위/지속/안전은 "보상 종류"가 아니라 스킬 위력 체감과 운용 방식을 설명하는 공용 성장축이다.
드롭 줍기, 중간 상점 구매, 경험치 기반 레벨업은 이후 콘텐츠 확장 시 추가 검토한다.

### 새 스킬 vs 강화의 판단

시너지 시스템이 아직 없더라도 아래 트레이드오프로 충분한 판단 공간을 만든다.

- **새 스킬 추가**: 대응 범위와 역할 커버리지는 늘어나지만, 에이스 스킬의 재등장 주기는 길어진다
- **기존 스킬 강화**: 덱은 얇게 유지되지만, 나올 때마다 더 확실한 임팩트를 만든다

즉 MVP의 재미는 "조합 완성"보다
**덱 두께 vs 품질**, **역할 분화**, **프렌들리 파이어 리스크 관리**에서 나온다.

### 시너지의 위치

스킬 간 명시적 시너지는 Phase 2 과제다.
스킬 수와 태그 풀이 충분히 늘어난 뒤, 화염/근접/원거리/구조 같은 조합 규칙을 붙여
조합 발견형 덱빌딩으로 확장한다.

### 메타 성장

존재. 장기 플레이 동기와 반복 플레이 보상을 제공한다. 상세 미정.

## 콘텐츠

| 항목 | 내용 |
|---|---|
| 캐릭터 | 단일 캐릭터 + 빌드 차별화 |
| 적 구성 | 잡몹 + 위험 몹 혼합 |
| 보스 | 존재 |

### 한 판 흐름

1. 방 생성/입장
2. 초기 스킬 선택 또는 기본 스킬 지급
3. 웨이브 시작
4. 전투
5. 필드 드롭 획득 (힐, 짧은 버프, 즉시 효과)
6. 웨이브 클리어
7. 성장 선택 (3지선다: 새 스킬 또는 보유 스킬 강화)
8. 다음 웨이브
9. 최종 보스 또는 마지막 웨이브 생존
10. 결과 화면

## 사망 처리

**다운 후 구조 가능** (추천 기본값)

이유:
- 프렌들리 파이어로 아군을 다운시키고 → 직접 구조하러 가는 상황이 게임 정체성에 맞음
- 구조 중에도 적이 오니까 긴장감 유지
- 코옵성, 짧은 세션, 카오스 상황과 가장 잘 맞음

## 톤

- 완전 경쟁 PvP는 아님
- 완전 안전한 코옵도 아님
- 실수와 혼돈이 허용되는 협력 생존 게임
