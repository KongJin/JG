# Garage UI Design v2

이 문서는 매치 전 편성 화면(차고)의 UX/UI를 정의한다.
`unit_module_design.md`와 `module_data_structure.md`의 하위 문서다.

**상위 계획 의존 문서**:
- `../../agent/game_design.md` — 게임 디자인 SSOT ("가벼운 차고", 코옵 PvE)
- `implementation_plan_redesign_execution.md` — 로비 편성 단계 분리 전략
- `implementation_plan_unit_module.md` — 네트워크 계약, CustomProperties 소유권
- `../../agent/state_ownership.md` — CustomProperties 소유권 맵

---

## 전제: 코옵 로비 플로우 정렬

이 문서는 다음 상위 결정사항을 준수한다.

### 1. 차고는 로비 내부의 편성 패널이다

- 차고는 독립된 화면이 아니라 **로비 씬 안에 임베드된 패널**이다.
- 편성 완료는 **리드니스 토글**("편성 완료" 버튼)로 표현된다.
- 전투 진입은 **호스트의 시작 버튼**으로만 가능하다.

### 2. 네트워크 상태가 메인 데이터 경로다

- CustomProperties["garageRoster"]가 실제 전투 진입용 데이터.
- CustomProperties["garageReady"]가 리드니스 상태.
- 로컬 JSON 저장/불러오기는 **보조 캐시** (이전 세션 복원용 편의 기능).

### 3. MVP는 전부 해금 상태

- 게임 디자인 SSOT: "기본 유닛과 기본 모듈은 너무 강하게 잠그지 않는다."
- MVP에서는 모든 프레임과 모듈이 **기본 해금**이다.
- 잠금/해금 UI는 넣지 않는다.

### 4. 모바일 퍼스트: 탭 중심 상호작용

- MVP에서는 **탭**만 사용한다.
- 드래그 앤 드롭, 호버, 스와이프는 제거한다.
- 편성 순서 변경은 MVP에서 지원하지 않는다 (고정 순서).

## 개요

### 차고의 역할

차고는 플레이어가 **전투 전 유닛을 편성하고 모듈을 조합하는 공간**이다.
여기서 보내는 시간이 전투만큼 중요해야 한다 (게임 디자인 SSOT).

### 핵심 목표

1. **직관적 조합**: 상/중/하단 3슬롯 구조가 한눈에 보여야 한다
2. **실시간 피드백**: 모듈 선택 시 스탯, 비용, 역할이 즉시 반영되어야 한다
3. **유효성 체크**: 금지조합은 사전에 경고되어야 한다
4. **코옵 가시성**: 룸원들의 편성 완료 상태가 한눈에 보여야 한다
5. **모바일 우선**: 탭 중심 상호작용, 세로 화면 지원

## 상세 UI 컴포넌트

### 1. 룸원 편성 상태 패널

```
┌──────────────────────────────────────┐
│  💡 룸원 편성 상태                    │
│                                      │
│  ┌──────┐ ┌──────┐ ┌──────┐         │
│  │ 🟢나  │ │ 🟡친구1│ │ 🟢친구2│  →      │
│  │ ✅완료│ │ ⏳편성중│ │ ✅완료 │         │
│  └──────┘ └──────┘ └──────┘         │
└──────────────────────────────────────┘
```

**상태 표시 규칙**:

| 상태 | 아이콘 | 색상 | 의미 |
|---|---|---|---|
| 편성 완료 | ✅ | 🟢 초록 | `garageReady == true` |
| 편성 중 | ⏳ | 🟡 노랑 | `garageReady == false`, 편성 있음 |
| 미접속 | ⚪ | 회색 | 빈 슬롯 |

**동작**:
- 다른 플레이어의 상태는 **읽기 전용** (탭해도 내 편성 안 열림)
- 내 상태만 탭 가능 → 내 편성 수정 화면으로 전환
- 호스트만 "시작" 버튼이 별도로 표시됨 (모두 ✅일 때 활성화)

**호스트 시작 버튼 (호스트만 표시)**:

```
┌──────────────────────────────────────┐
│  [🚀 전투 시작]                       │  ← 모두 ✅일 때 활성화
│  (0/4명 준비 안 함)                   │  ← 아닌 경우 비활성화 + 카운트
└──────────────────────────────────────┘
```

### 5. 실시간 스탯 표시

```
┌──────────────────────────────────────┐
│  조합 결과                            │
│                                      │
│  HP:       ████████░░░░░░  800       │
│  공격력:   ████░░░░░░░░░░  30        │
│  공격속도: █████░░░░░░░░░  1.0/s     │
│  사거리:   █████░░░░░░░░░  4m        │
│  이동범위: ███░░░░░░░░░░░  3m        │
│                                      │
│  💰 소환 비용: 52                     │
│  🎭 예상 역할: 탱커                   │
└──────────────────────────────────────┘
```

**중요**: **"총 비용"은 표시하지 않는다.**
- 게임 디자인 SSOT는 유닛별 소환 비용만 정의한다.
- 편성 총합에 상한선이나 게임적 의미가 없다.
- 개별 유닛 비용만 모듈 조합 패널 하단에 표시한다.

**동작**:
- 모듈 선택 시 **즉시 업데이트** (애니메이션: 0.2초 트윈)
- 바 색상은 역할별 테마:
  - 탱커: 🔵 파랑
  - 근접딜러: 🔴 빨강
  - 원거리: 🟢 초록
  - 지원: 🟡 노랑

**비용 표시 규칙 (개별 유닛)**:
- 15~30: 🟢 저렴 (초록)
- 31~50: ⚪ 보통 (흰색)
- 51~70: 🟠 비쌈 (주황)
- 71~80: 🔴 매우 비쌈 (빨강)

#### 정상 상태

```
┌──────────────────────────────────────┐
│  ✅ 유효한 조합입니다.                 │
│  역할: 탱커 (전선 고정)               │
└──────────────────────────────────────┘
```

#### 금지조합 감지 시

```
┌──────────────────────────────────────┐
│  ⚠️ 이 조합은 사용할 수 없습니다!      │
│                                      │
│  이유: 연사(사거리 3m) + 고정포대    │
│        (이동범위 2m)                 │
│        → 적에게 접근할 수 없습니다.  │
│                                      │
│  💡 이동범위가 3m 이상인 모듈을       │
│     선택하거나, 사거리가 짧은         │
│     화력 모듈로 변경해보세요.         │
└──────────────────────────────────────┘
```

**동작**:
- 금지조합 선택 시 **즉시 경고** (빨간 테두리 + 메시지)
- "완료" 버튼 비활성화 (모듈 조합 패널 내에서만)
- 편성 슬롯에 추가 불가

## 사용자 상호작용 플로우

### 시나리오 1: 첫 유닛 편성

```
1. 로비 진입 → 차고 패널 자동 표시
2. 빈 슬롯(＋) 탭
3. 프레임 선택 팝업에서 "가디언" 탭
4. 기본 모듈 조합 자동 적용
   - 상단: 단일탄 (기본값)
   - 중단: 철벽 (고정)
   - 하단: 중장갑 (기본값)
5. 스탯/비용 확인
6. 슬롯 1번 채워짐 → 편성 수 1/3
```

### 시나리오 2: 모듈 조합 변경

```
1. 편성 슬롯에서 유닛 탭
2. 모듈 조합 패널 하단 시트로 열림
3. "화력 모듈" 탭 → 모듈 선택 시트 열림
4. "광유탄" 탭
5. 스탯 즉시 업데이트
6. ✅ 경고: "이동범위 3m + 사거리 6m = 유효"
7. "완료" 탭 → 변경 반영, 시트 닫힘
```

### 시나리오 3: 금지조합 감지

```
1. 하단 모듈에서 "고정포대" 탭 (이동범위 2m)
2. 상단 모듈이 "연사" (사거리 3m)인 상태
3. ⚠️ 즉시 경고 표시
   - 빨간 테두리
   - "이 조합은 사용할 수 없습니다" 메시지
4. 하단 → "경량"으로 변경 (이동범위 7m)
5. ✅ "유효한 조합입니다" 메시지로 변경
6. "완료" 활성화
```

### 시나리오 4: 편성 완료 → 리드니스

```
1. 3기 이상 편성 완료
2. "편성 완료" 버튼 활성화 → 탭
3. CustomProperties["garageReady"] = true
4. 룸원 상태 패널에서 내 상태 🟢로 변경
5. 버튼 텍스트 "수정"으로 변경 → 다시 탭하면 수정 모드
```

### 시나리오 5: 호스트 시작

```
1. 모든 플레이어 garageReady == true
2. 호스트의 "전투 시작" 버튼 활성화
3. 호스트 탭 → SceneManager.LoadScene("JG_BattleScene_Redesign")
4. CustomProperties["garageRoster"]에서 모든 플레이어 편성 복구
```

## 데이터 연동

### 차고 화면이 참조하는 데이터

```
GarageScreen (Presentation)
├── ModuleCatalog (ScriptableObject - Infrastructure)
│   ├── UnitFrames
│   ├── FirepowerModules
│   └── MobilityModules
├── IGarageNetworkPort (Application Port)
│   ├── SyncRoster(roster)  → CustomProperties["garageRoster"]
│   ├── SyncReady(ready)    → CustomProperties["garageReady"]
│   └── GetOtherPlayersRoster() → 룸원 편성 읽기
├── GarageJsonPersistence (Infrastructure - 보조 캐시)
│   ├── Save(roster) → Application.persistentDataPath
│   └── Load() → 이전 세션 편성 복원
└── UnitComposition (Domain)
    ├── Compose(frame, firepower, mobility)
    ├── Validate(frame, firepower, mobility)
    └── CostCalculator
```

### 화면 진입 시

1. `GarageJsonPersistence.Load()` → 이전 편성 복원 (보조)
2. `ModuleCatalog` → 사용 가능한 프레임/모듈 로드 (MVP 전체 해금)
3. 편성 슬롯에 이전 조합 복원
4. 복원된 편성이 있으면 CustomProperties에도 동기화

### 화면 이탈 시 (편성 완료 버튼)

1. `GarageRoster.IsValid` 체크 (3~5기)
2. 금지조합 검증
3. `IGarageNetworkPort.SyncRoster(roster)` → CustomProperties["garageRoster"]
4. `IGarageNetworkPort.SyncReady(true)` → CustomProperties["garageReady"]
5. `GarageJsonPersistence.Save(roster)` → 보조 캐시 백업
6. 버튼 텍스트 "수정"으로 변경

### 편성 수정 시

1. "수정" 버튼 탭 → CustomProperties["garageReady"] = false
2. 룸원 상태 패널에서 내 상태 🟡로 변경
3. 편성 수정 가능
4. 다시 "편성 완료" → SyncReady(true)

## 모바일 UX 가이드라인

### 상호작용 통일 (MVP)

| 동작 | 방식 |
|---|---|
| 모듈 선택 | 탭 → 하단 시트 |
| 프레임 선택 | 탭 → 팝업 리스트 |
| 유닛 상세 보기 | 길게 누르기 (0.5초) → 액션 시트 |
| 유닛 제거 | 길게 누르기 → [제거] 탭 |
| 편성 완료/수정 | 버튼 탭 (토글) |
| 드래그 앤 드롭 | ❌ MVP 미지원 |
| 호버 | ❌ 모바일에서 무의미 |
| 스와이프 제거 | ❌ MVP 미지원 |

### 터치 타겟 크기

- 최소 44×44px (Apple HIG 기준)
- 모듈 선택 버튼: 60px 높이 권장
- 프레임 카드: 80×100px 권장
- "편성 완료" 버튼: 60px 높이, 화면 가로 fullWidth

### 스크롤 동작

- 프레임 선택 팝업: 세로 스크롤
- 모듈 선택 시트: 세로 스크롤 (항목 많을 시)
- 전체 화면: 세로 스크롤 가능 (레이아웃 길 경우)
- 하단 시트: 드래그 다운으로 닫기 가능

### 애니메이션/피드백

| 이벤트 | 효과 |
|---|---|
| 모듈 선택 | 스탯 바 0.2초 트윈 애니메이션 |
| 비용 변경 | 숫자 카운트업/다운 (0.3초) |
| 금지조합 감지 | 패널 붉은 테두리 + 진동 (0.1초) |
| 유효조합 확인 | 초록 체크 표시 + 부드러운 강조 |
| 편성 추가 | 슬롯에 카드 슬라이드 인 (0.3초) |
| 편성 제거 | 카드 슬라이드 아웃 (0.3초) |
| 하단 시트 열림 | 아래에서 슬라이드 업 (0.3초) |
| 하단 시트 닫힘 | 아래로 슬라이드 다운 (0.2초) |
