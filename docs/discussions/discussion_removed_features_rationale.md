이 문서는 MVP 범위에서 **제외했던 기능의 근거**만 짧게 남긴 reference다. 현재 제품 판단의 SSOT는 [`game_design.md`](../design/game_design.md) 이다.

---

## 정리 원칙

- MVP에 넣지 않았던 이유만 적고, 수치나 구현 규칙은 다시 정의하지 않는다.
- 범위가 다시 바뀌면 이 문서보다 `game_design.md`를 먼저 갱신한다.

---

## 제외 항목과 근거

### 다운/구조 중심 코옵 드라마

- 초기 토론에서는 강한 팀 서사 후보였지만, 첫 MVP에서는 덱 순환과 유닛 소환 판단을 더 우선 검증하기로 했다.
- 실시간 혼전에서 다운/구조 규칙까지 넣으면 첫 세션 인지 부하가 커지고, 프렌들리 파이어·밀도 문제와 함께 디버깅 범위가 급격히 넓어진다.
- 이 근거는 `game_design.md`의 소환/편성 중심 전투 루프 기준을 따른다.

### 웨이브 사이 보상 선택 / 런 중 드래프트

- 구형 액션 루프의 성장 감각은 좋았지만, 현재 방향은 “연속 압박 속에서 어떤 유닛을 언제 꺼낼지”를 먼저 검증하는 쪽으로 옮겨 갔다.
- 웨이브 종료 보상 선택을 유지하면 전투 리듬보다 보상 UI와 풀 구성이 먼저 병목이 된다.
- 이 근거는 `game_design.md`의 연속 압박 기반 구조와 차고/편성 중심 판단을 따른다.

### 복잡한 메타 화폐 / 강화 트리 / 카드 수집 중심 메타

- 장기 리텐션 장치로는 의미가 있지만, MVP 단계에서는 한 판의 판단 재미가 먼저 검증되어야 한다.
- 복잡한 메타 구조를 먼저 넣으면 한 판 재미보다 누적 보상 구조가 주연이 된다.
- 이 근거는 `game_design.md`의 “가벼운 차고”와 최소 메타 전제를 따른다.
