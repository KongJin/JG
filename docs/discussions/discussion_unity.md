  2. rule-harness의 반복 실패 💥

  advisory-memory.json에서 볼 수 있듯이:

  static-scan-failed: 14회 (hitCount: 14)
  └─ Before=8 After=7 Resolved=0 Expected=1
  └─ 마지막: 2026-04-16

  harness-stage-failed: 여러 회
  └─ 네트워크 오류 (중국어 오류 메시지)
  └─ 타임아웃 (120초)

  - 정적 스캔이 계속 실패하고 해결되지 않음
  - 네트워크/타임아웃 문제로 진단 단계가 자주 실패
  - 이는 규칙 시스템이 실제로 작동하지 않고 있음을 시사

  ---
  3. 규칙과 실제 코드의 일치 확인 어려움 🔍

  코드에서 검색해 보면:
  - GetComponent() 사용: 0건 (Features에서)
  - FindObjectOfType: 0건 (Features에서)
  - AddComponent: 0건 (Features에서)

  하지만 FromStore/Editor/Shared에는 141건의 GetComponent 사용이 있음:
  Assets/FromStore/Photon/**: 20건+
  Assets/Editor/UnityMcp/**: 10건+

  - 문제: 외부 라이브러리(Photon, DOTween)와 Editor 스크립트는 규칙이 적용되지 않음
  - 한계: 허용 예외와 실제 코드 사이의 명확한 경계가 없음

  ┌────────────────┬──────────────────────────────────────────────────────────────────────┐

  ├────────────────┼──────────────────────────────────────────────────────────────────────┤

  ├────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ 허용 예외      │ "Unity/Photon이 해당 사례를 inspector에서 연결할 수 없어서" — 주관적 │
  ├────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ enum 기반 분기 │ "명령 디스패치와 단순 값 매핑은 switch 허용" — 경계 불분명           │
  ├────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ 런타임 탐색    │ "동일 GameObject 내부 helper 탐색만" — helper의 정의 불확실          │

  - 규칙 적용 범위가 모호하면 개발자가 주관적으로 판단해야 함
  - 코드 리뷰에서 일관성이 깨질 수 있음

  ---
  6. CLI vs MCP 결정 트리의 현실성 ⚡

  컴파일 에러 체크 → CLI (또는 MCP 둘 다 가능)

  실제 상황:
  - Unity Editor가 꺼져 있으면 CLI로만 가능
  - Unity Editor가 켜져 있으면 MCP로도 가능
  - 하지만 MCP가 항상 사용 가능한 상태인지 확인 필요
  - 문제: 결정 트리가 현재 상태를 고려하지 않음
  - GET /health로 bridge 상태 확인이 필요하지만 문서에 기본 순서로 명시되지 않음

  7. 규칙 업데이트 및 승인 프로세스 부재 📋

  - 규칙이 언제, 어떻게 업데이트되는지 명시되지 않음
  - 누가 승인하는지 절차가 없음
  - promotionCandidates가 있지만 실제로 승격되는 사례가 없음

  ---
  8. MCP 핸들러 템플릿과 실제 구현의 격차 🔧

  표준 템플릿은 명확하지만:
  - 모든 핸들러가 이 템플릿을 따르는지 확인 방법이 없음
  - 새로운 핸들러 추가 시 템플릿을 강제할 수 없음
  - linter/formatter가 없으므로 실시간 피드백 불가

  ---
  9. 규칙 위반 감지 능력 부족 🚨

  실제 코드에서 규칙 위반을 감지하는 방법:
  1. grep으로 패턴 검색
  2. 수동 코드 리뷰
  3. rule-harness 실행 (하지만 계속 실패)

  - 자동화된 linter가 없음
  - IDE 통합이 없음 (Roslyn analyzer 등)
  - CI/CD에서 실시간 검증이 안됨
